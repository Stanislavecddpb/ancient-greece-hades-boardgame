# План реализации дополнения «Киклады: Аид» (Cyclades: Hades)

Документ описывает, как навесить дополнение Hades на текущую кодовую базу
(boardgame.io, монорепо `engine` / `server` / `client`). Источник правил —
`rules_cyclades-hades.pdf` (4 модуля + новые компоненты).

## Как устроена база (точки расширения)

- **`packages/engine`** — вся логика. Состояние партии — объект `G`
  (`CycladesState` в `types.ts`). Ходы объявлены в `game.ts` внутри фаз
  `lobby → auction → actions → (петля)`. Каждый ход — тонкая обёртка, которая
  валидирует и вызывает «эффект» из отдельного модуля (`actions.ts`,
  `auction.ts`, `creatures.ts`, `movement.ts`, `combat.ts`, `metropolis.ts`,
  `income.ts`).
- **Существа** — реестр `CREATURES` в `creatures.ts`: у каждого `id`, `cost`,
  `target`, `desc`, флаг `placed` (фигура на доске) и функция `apply`. Колода —
  `makeCreatureDeck()`, рынок — `CreatureMarket`.
- **Арт карт** — клиент грузит `/<gods|creatures>/<id>.jpg` из
  `packages/client/public/`, а при отсутствии файла показывает текстовый/эмодзи
  фолбэк (`onError`). **Значит, для новых карт достаточно завести `id` и
  текстовый фолбэк в коде — а `.jpg` ты докинешь руками** (как в базовой игре).
- **Юниты** сейчас — просто числа `troops`/`fleets` на территории и
  `troopsSupply`/`fleetsSupply` у игрока. Отдельного «типа» юнита нет — это
  главное, что придётся расширить под Нежить.

## Модель данных: что добавить в `CycladesState` (`types.ts`)

```ts
// Нежить — отдельный тип юнита (не даёт золота Некрополю, теряется в выбранном
// игроком порядке). Храним параллельно обычным troops/fleets.
interface Island  { /* + */ undeadTroops: number; necropolis?: boolean; theater?: boolean; prosperity: number; }
interface Sea     { /* + */ undeadFleets: number; prosperity: number; }
interface PlayerData { /* + */ undeadTroopsSupply: number; undeadFleetsSupply: number;
  heroes: HeroInstance[]; priestesses: number; magicItems: string[]; }

interface HeroInstance { id: string; islandId: TerritoryId; recruitedCycle: number; }

// Модуль 2 — трек Аида
interface HadesTrack { column: number; active: boolean; replacedGod: GodName | null; }

// верхнего уровня в G:
hades: HadesTrack;                 // трек угрозы (Модуль 2)
necropolisGold: number;            // ЗМ, накопленные на Некрополе
hadesRecruit: HadesRecruitState | null;   // активный наём Нежити
hadesMove: HadesMoveState | null;          // активное перемещение с Нежитью
favor: { deck: string[]; current: string | null };  // Модуль 4
priestessDeck: number;             // запас карт Жриц (просто счётчик)
itemDeck: string[];                // колода Волшебных Предметов
heroUpkeepPending: PlayerID | null; // оплата 2🪙/Героя после дохода
```

Все новые поля инициализируются в `setup.ts`. Колода существ (Модуль 3)
расширяется новыми картами в `makeCreatureDeck()`.

## Модуль 2 — Аид и его Нежить  (ядро дополнения, делать первым)

**Трек угрозы.** Перед фазой `auction` каждого цикла (в `auction.onBegin`, до
`setupAuction`) бросаем 2 кубика и двигаем `hades.column`. Если столбец достигает
или проходит «9» — Аид входит в игру: `hades.active = true`, тайл Аида накрывает
бога ровно над Аполлоном (тот бог в этом цикле заменяется на Аид).

**Реализация замены бога.** В `setupAuction` после раскладки богов помечаем
верхний слот (над Аполлоном) как Аид (`slot.god = 'hades'` либо отдельный флаг
`slot.isHades`). Победитель этого слота в фазе действий получает действия Аида
вместо действий обычного бога.

**Действия Аида** (новые ходы в `game.ts`, эффекты в новом `hades.ts`):
- `recruitUndead(troops, fleets)` — 1 Нежить бесплатно, далее 1/2/3/4🪙; не больше
  4 за ход; комбинация Войск+Флотилий. Размещение на своих островах/морях.
- `buildNecropolis(islandId)` — на месте Метрополии своего острова (сносит то, что
  там было). На поле только один Некрополь; при повторном приходе Аида любой игрок
  может перенести тайл к себе (ЗМ остаются на старом острове).
- `moveWithUndead(...)` — переиспользует логику Посейдона/Ареса из `movement.ts`,
  но **в каждом перемещении обязана участвовать ≥1 Нежить**; живые могут
  присоединяться. При проигрыше боя игрок сам выбирает порядок потерь (Нежить или
  обычные) — расширить `combat.ts`/`CombatState` опцией выбора потерь.

**Эффект Некрополя.** В `combat.ts` и в эффектах существ: когда гибнет ОБЫЧНОЕ
(не Нежить) Войско/Флотилия — `necropolisGold += 1`. В `income.ts` владелец
острова с Некрополем забирает накопленные ЗМ в фазе дохода.

**Конец цикла с Аидом** (`endCycle` в `actions.ts`): убрать всю Нежить с доски и
из supply вернуть, `hades = {column:0, active:false, replacedGod:null}`, тайл
перевернуть на «Угрозу».

**Клиент:** трек Аида (колонна 0–9) над полем богов; слот Аида в `GodBoard`;
панель действий Аида (наём Нежити с ценником, кнопка Некрополь, перемещение);
отрисовка фигур Нежити и Некрополя на `BoardMap`/`Board`.

## Модуль 3 — Герои и новые Мифические Существа

**Новые существа** (добавить в `CREATURES`, замешиваются в общую колоду):
- `cerberus` (фигура, есть миниатюра) — как Минотавр/Хирон: ставится на остров, в
  фазе дохода владелец собирает доход с этого острова вместо хозяина (вкл. ЗМ
  Некрополя). Уже частично есть инфраструктура `placed`-существ.
- `furies` — взять маркер процветания с любого острова на свой (печатные рога не
  трогаются).
- `empusa` — украсть все ЗМ с Некрополя.
- `charon` — вернуть в игру одного выбывшего Героя (кроме Пентесилеи, ушедшей на
  свой остров).
- `chiron_hades` — заменяет базового Хирона: защищает ещё и от Эмпусы, Фурий и
  самопожертвования Персея. (Заменить запись `chiron`, не держать обе.)

**Герои** — новый класс карт (НЕ существа). Реестр `HEROES` в новом `heroes.ts`:
у каждого `id`, `name`, боевая мощь (модификатор в бою) и «сила
самопожертвования» (эффект при жертве в фазе действий). Замешиваются в колоду
существ, но:
- Храмы НЕ дают скидку; Зевсом не сбрасываются; на острове можно держать
  несколько Героев и вместе с существом.
- При найме — фигура на свой остров + запись в `player.heroes`.
- **Апкип:** в начале цикла сразу после дохода — 2🪙 за каждого Героя, иначе Герой
  исчезает навсегда (карта из игры, не в сброс). Новое состояние
  `heroUpkeepPending` + ход `payHeroUpkeep` / `dismissHero`.
- **Бой:** Герой считается за 1 Войско + даёт боевую мощь — расширить `combat.ts`,
  чтобы учитывать Героев на острове.
- **Самопожертвование:** ход `sacrificeHero(heroId, ...)` — нельзя в тот же ход,
  когда нанят, и нельзя если ставил на Аполлона; нельзя если Герой убит в бою.

Конкретные Герои: Ахиллес (×2 в бою; жертва — Метрополия при 4 островах), Улисс
(атакуя, защитник без бонуса крепости/метрополии; жертва — Метрополия за
Порт+Храм+Универ), Гектор (−1 войско у атакующего; жертва — философы за жрецов),
Персей (отступление вместо гибели; жертва — увод войск на незащищённый остров),
Мидас (1🪙 = переброс костей; жертва — Метрополия за 15🪙), Пентесилея (победа при
ничьей; жертва — Метрополия на секретном непобедимом острове).

## Модуль 4 — Божественное Покровительство

**Колоды:** Жрицы (счётчик `priestessDeck`), Волшебные Предметы (`itemDeck`),
тайлы Покровительства (`favor.deck`).

**Жрицы** — сбросить карту Жрицы, чтобы (а) не платить апкип Героя, или (б) не
сбрасывать фигуру существа и продлить её на ход. Ходы `usePriestessForHero`,
`usePriestessForCreature`.

**Покровительство.** В фазе `auction.onBegin`: если Аид активен — перетасовать
тайлы в новую колоду (Покровительство не используется); иначе открыть верхний
тайл рядом с активным богом над Аполлоном. Победитель того бога может опционально
забрать и Покровительство (бонусный бог). В конце фазы действий тайл сбрасывается.
Каждый тайл = карта Жрицы/Предмет + разовая способность.

Боги-покровители: Артемида, Афродита, Дионис, Деметра, Гестия, Гера, Гефест,
Гермес (эффекты — см. правила, реализуются как разовые ходы).

**Волшебные Предметы** (10 шт.) — копятся перед игроком, разовые: Стрелы Артемиды
(убрать фигуру существа), Ящик Пандоры (украсть Жрицу), Солнечная колесница
(перегруппировка войск), Ожерелье гармонии (иммунитет к атакам на ход), Трезубец
Посейдона / Стрелы Аполлона (доп. кубик в бою), Пояс Афродиты (запрет ставки),
Щит Зевса (переброс кубика Аида), Дворец на горе Олимп (порядок богов), Скипетр
Пелопа (вытянуть существо с фигурой).

**Новые постройки:**
- **Некрополь** (Модуль 2) — на месте Метрополии.
- **Театр** — «джокер»-постройка, заменяет любое здание для Метрополии; без
  собственного эффекта. Добавить `BuildingType` учёт театра в `metropolis.ts`
  (Театр закрывает любой недостающий тип). Гигант не может его разрушить (только
  мелкие постройки) — а Некрополь Гигант тоже не сносит.
- **Маркеры процветания** — лежат на островах/морях, +1 к доходу (как рога, но
  переносимые Фуриями/Гермесом). Поле `prosperity` на территориях, учёт в
  `income.ts`.

## Модуль 1 — Свободное расположение  (опциональный стартовый режим)

Альтернативная подготовка: предварительный цикл ставок за стартовые острова, по
7🪙, без стартовых войск/флота на поле. Реализуется как флаг режима в `setup.ts` +
особая стартовая фаза перед `auction`. **Рекомендую делать последним** — он
самодостаточен и не блокирует остальные модули. Сделать переключателем в лобби
(как и каждый модуль — включаемым отдельно, по духу правил).

## Рекомендованный порядок работ

1. **Каркас модели** — добавить поля в `types.ts` + инициализация в `setup.ts`
   (Нежить, Некрополь/Театр/процветание, герои, колоды). Без логики, чтобы
   проект собирался и тесты шли.
2. **Модуль 2 (Аид)** целиком: трек угрозы, замена бога, действия Аида, Нежить,
   Некрополь, конец цикла. Это половина дополнения и самостоятельно играбельно.
3. **Постройки/процветание** (Театр, маркеры) — нужны нескольким картам.
4. **Модуль 3**: новые существа → затем Герои (апкип, бой, самопожертвование).
5. **Модуль 4**: Жрицы → тайлы Покровительства → Волшебные Предметы.
6. **Модуль 1** (свободное расположение) — стартовый режим.
7. **Клиент и арт-плейсхолдеры** ведём параллельно каждому модулю; финально —
   подмена картинок (твои файлы) и UI-полировка.

Каждый модуль — включаемый отдельно (лобби), как и рекомендуют правила. Под
движок есть тесты (`vitest`) — новую логику покрываем по образцу
`creatures.test.ts` / `rules.test.ts`.
